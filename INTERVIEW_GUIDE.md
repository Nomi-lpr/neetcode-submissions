# 面试手撕代码：你到底在被考察什么

刷题不是背题。面试官在 45 分钟里真正评估的是下面这些素质，刷每一道题时都应该有意识地训练它们。

## 一、面试的标准流程（每道题都按这个演练）

1. **复述与澄清（2 分钟）**：用自己的话复述题意，主动问澄清问题——
   输入范围多大？会有空输入吗？有重复元素吗？返回下标还是值？可以修改原数组吗？
   *面试官最反感的行为：拿到题闷头就写。*
2. **先给暴力解**：口头说出最直接的解法和它的复杂度（"暴力是双重循环，O(n²)"）。
   这证明你至少能解出问题，也为优化提供了对比基准。
3. **找瓶颈、引出优化**："O(n²) 的瓶颈在于每次都重新查找 → 用 hashmap 把查找降到 O(1)"。
   优化几乎总是"用空间换时间"或"利用数据的某种性质（有序、单调、范围有限）"。
4. **写码前先说方案，获得认可再动手**。写码时边写边讲（think aloud）。
5. **写完主动测试**：不等面试官提醒，自己走一遍普通用例 + 至少一个边界用例，
   逐行 trace 变量的值。
6. **主动报复杂度**：写完说"时间 O(n)，空间 O(n)"，并简述原因。

## 二、复杂度直觉（必须内化的对照表）

| 数据规模 n | 可接受的复杂度 |
|---|---|
| n ≤ 20 | O(2ⁿ)（回溯、状压） |
| n ≤ 10³ | O(n²) |
| n ≤ 10⁵ ~ 10⁶ | O(n log n) 或 O(n) |
| n ≥ 10⁷ | O(n) 或 O(log n) |

看到题目给的数据范围，就应该能反推面试官期待的复杂度，从而缩小套路搜索范围。
另外记住：排序 O(n log n)；heap 的 push/pop O(log n)；dict/set 的增删查 O(1) 均摊；
list 头部插入/删除是 O(n)（要用 deque）。

## 三、边界条件的固定检查清单

每写完一道题，按这个清单过一遍（这也是笔记模板里"边界条件清单"的来源）：

- 空输入 / None；单个元素；两个元素
- 所有元素相同；已经有序 / 逆序
- 负数、零、极大值
- 重复元素（会不会导致 set 去重丢信息？）
- 字符串：空串、单字符、全同字符、大小写
- 链表/树：空节点、单节点、只有左/右子树
- 下标：off-by-one，`range(len(a)-1)` 还是 `range(len(a))`，循环里 `i+1` 会不会越界

## 四、题型 → 套路的映射（刷题真正要建立的东西）

刷 NeetCode 150 的目的不是记住 150 道题，而是记住 ~15 个套路以及触发它们的信号词：

| 信号 | 套路 |
|---|---|
| "找两个数/是否存在/统计出现次数" | hashmap / set |
| 有序数组 + 找目标 | 二分 / 双指针 |
| "连续子数组/子串" + 最长/最短/计数 | 滑动窗口 |
| 括号匹配、最近相关性、撤销 | 栈 |
| "下一个更大元素"类 | 单调栈 |
| 第 K 大/小、流式数据 | heap |
| 前缀和能否表达 | prefix sum + hashmap |
| 网格/图的连通、层级 | BFS / DFS |
| "最少步数" | BFS |
| 决策序列、最优子结构、"多少种方式" | DP |
| 枚举所有组合/排列 | 回溯 |
| 区间调度/重叠 | 排序 + 贪心/扫描 |

复习时的自测方式：只看题目名，先说出套路和复杂度，再动手。说不出套路比写不出代码更危险。

## 五、Python 手撕代码最常用的武器库

面试用 Python 的优势就是这些内置结构，务必熟到不假思索：

```python
# dict：计数与查找
count = {}
count[x] = count.get(x, 0) + 1          # 不存在则取默认值
from collections import defaultdict, Counter
count = Counter(nums)                    # 一行计数
graph = defaultdict(list)                # 邻接表不用判断 key 是否存在

# set：O(1) 存在性判断、去重
seen = set(); x in seen; seen.add(x)

# list 常用
a[::-1]           # 反转（新列表）
sorted(a, key=lambda p: p[1])            # 按自定义键排序
for i, v in enumerate(a): ...            # 同时要下标和值
zip(a, b)         # 并行遍历

# deque：O(1) 双端操作（BFS 队列必用，别用 list.pop(0)）
from collections import deque
q = deque([start]); q.popleft(); q.append(x)

# heap（最小堆；要最大堆就存负数）
import heapq
heapq.heappush(h, x); heapq.heappop(h)

# 常见陷阱
[[0]*n for _ in range(m)]   # 正确的二维数组；[[0]*n]*m 是同一行的引用！
a = b                        # 这是引用不是拷贝；要 a = b[:] 或 list(b)
```

## 六、复习节奏建议

- 用仓库根目录的 `PROGRESS.md`：遮住"一句话思路"列，看题名回忆套路 → 核对
- 间隔重复：新题在第 1、3、7、21 天后各重做一次（只看题名重写，不看笔记）
- 每道题的 NOTES.md 里"面试口述剧本"要**开口念出来**，口述能力和写码能力是分开训练的
- 一道题卡 30 分钟以上就看题解，但看完必须合上题解自己重写一遍才算完成
