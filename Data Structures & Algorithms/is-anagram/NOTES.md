# is-anagram

## 一句话思路
用长度为 26 的数组分别统计两个字符串中字母出现次数，再判断每个字母的出现次数是否全部为 0。

## 复杂度分析
- **时间复杂度**：O(n)，其中 n 是字符串长度。
  - 首先判断长度是否相等 O(1)。
  - 第一个 for 循环遍历 s 和 t 各一次，共 O(n)。
  - 第二个 for 循环遍历 26 个字母，O(26)，视为常数时间。
- **空间复杂度**：O(1)（因为计数数组大小始终固定为 26 跟输入无关）。
- **能否更优**：这已经是最优了。即使是排序法，时间复杂度也至少 O(nlogn)，而本解法只需 O(n)。所有方法至少要把每个字符看一遍，因此时间复杂度下限是 Ω(n)。

## 边界条件清单

| 边界情况           | 例子           | 代码是否覆盖 | 说明                                          |
|-------------------|---------------|--------------|-----------------------------------------------|
| 空字符串           | s="", t=""    | 覆盖         | 长度相等，直接进入 0 次循环，所有计数为 0，返回 True |
| 单个字符           | s="a", t="a"/"b" | 覆盖      | 长度相同继续判断，字母不同返回 False         |
| 不同长度           | s="ab", t="abc" | 覆盖        | 直接 return False                           |
| 重复字母           | s="aabb", t="bbaa" | 覆盖     | 统计次数后判断是否全为 0                     |
| 全部相同字母       | s="aaa", t="aaa" | 覆盖        | 统计次数后返回 True                         |
| 大写字母/非字母   | s="Abc", t="cab" | 未覆盖      | 只支持 'a'-'z' 的小写字母，若输入包含其他字符会出错或行为异常 |
| 非 ASCII/Unicode   | s="àbc", t="càb" | 未覆盖      | chr 不在 'a'-'z' 范围时，数组下标越界或出错  |

> **注意**：本解法只适用于 all lowercase letters，如面试时题意未指定要问清楚。

## Python 语法笔记

1. `count=[0]*26`
   - **作用**：创建长度为 26 的列表，所有元素初始化为 0，用于存 a-z 各字母计数。
   - **示例**：
     ```python
     arr = [0]*5   # [0, 0, 0, 0, 0]
     ```

2. `ord(s[i])-ord('a')`
   - **作用**：将字母 'a'-'z' 转换成 0-25 的索引，用于对应计数数组下标。
   - **示例**：
     ```python
     idx = ord('c') - ord('a') # 2
     ```

3. `for i in range(len(s)):`
   - **作用**：用下标遍历两个字符串，并行处理。
   - **最小变体**：
     ```python
     for i in range(len(a)):
         # 同步使用 a[i], b[i]
     ```

4. `for j in count:`
   - **作用**：直接遍历计数数组的每个元素。
   - **变体**：
     ```python
     for num in [1,2,3]:
         print(num)
     ```

## 面试口述剧本
1. 暴力做法是把字符串排序后比较，时间复杂度 O(nlogn)。
2. 我用一个长度为 26 的数组统计 s 每个字母出现次数加一、t 减一。
3. 如果 s 和 t 是 anagram，最后数组所有值一定都是 0。
4. 因为只遍历一遍字符串、并处理定长数组，时间复杂度为 O(n)，空间是 O(1)。
